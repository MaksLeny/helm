# 🎛️ Helm

**Локальный веб-пульт для управления компьютером с телефона.**

Helm превращает ваш телефон в пульт управления домашним ПК. Запускаете сервер на компьютере, открываете на телефоне веб-страницу в той же Wi-Fi сети — и управляете музыкой, смотрите загрузку системы, передаёте текст и файлы через буфер обмена, запускаете программы, открываете файлы и управляете окнами. Без установки приложений на телефон: всё работает через браузер.

Проект задуман для домашнего использования: один владелец-администратор (вы) и гости, которые иногда заходят в гости и которым удобно, например, переключить трек или скинуть ссылку на ваш ПК, не вставая с дивана.

## ✨ Что умеет Helm

Helm состоит из 7 разделов (каждый — отдельный плагин):

| Раздел | Что делает |
|--------|-----------|
| 🎵 **Мультимедиа** | Управление музыкой (Яндекс Музыка через системный плеер Windows): пауза, переключение треков, перемотка, громкость, обложка, лайк. Отдельный компактный плеер вынесен прямо на главный экран. |
| 📊 **Монитор** | Загрузка CPU, RAM, диска, сети, температура, батарея, время работы, список процессов с возможностью завершить любой. |
| ⚡ **Питание** | Сон, выключение, перезагрузка, блокировка ПК с телефона. |
| 🚀 **Приложения** | Запуск и закрытие программ из белого списка. Любимые можно отметить звёздочкой для быстрого доступа. |
| 📋 **Буфер обмена** | Двусторонний обмен текстом, ссылками, картинками и файлами между телефоном и ПК. История переживает перезагрузку компьютера. |
| 📁 **Файлы** | Навигация по дискам, скачивание файлов на телефон, загрузка с телефона, открытие файла на ПК в программе по умолчанию. |
| 🪟 **Окна** | Свернуть все окна, восстановить, переключиться между открытыми окнами. |

Дополнительно:
- 🎨 **6 цветовых тем** + светлый/тёмный режим, фон дашборда подкрашивается под обложку играющего трека.
- 📱 **PWA** — можно добавить на домашний экран телефона как приложение.
- 🔒 **Роли** — гости видят только разрешённые разделы, полный доступ под паролем.
- 🔔 **Уведомления** — когда гость заходит, на ПК всплывает уведомление.
- 👥 **Кто онлайн** и **лог действий** — видно, кто подключён и что делал.

## 🖼️ Как это выглядит

Главный экран: сверху плеер-виджет с обложкой и кнопками, ниже — плитки разделов. Всё адаптируется под размер экрана телефона.

## 💻 Требования

- **Операционная система:** Windows 10 / 11 (большинство функций завязано на Windows). На Linux/macOS сервер запустится, но разделы «Мультимедиа», «Питание», «Окна» работать не будут.
- **Python 3.12** — именно эта версия (библиотека `winsdk` для управления музыкой требует 3.12).
- **Телефон** в той же Wi-Fi сети, что и ПК.

## 📥 Установка

1. **Установите Python 3.12** с [python.org](https://www.python.org/downloads/). При установке обязательно отметьте галочку **«Add Python to PATH»**.

2. **Скачайте Helm** — нажмите зелёную кнопку `Code` → `Download ZIP` на странице репозитория, либо клонируйте:
   ```bash
   git clone https://github.com/MaksLeny/helm.git
   cd helm
   ```

3. **Установите зависимости.** Откройте командную строку (cmd) в папке проекта и выполните:
   ```bash
   pip install -r requirements.txt
   ```
   Это установит FastAPI, uvicorn и библиотеки для работы с музыкой, системой и т.д.

> 💡 Часть зависимостей опциональна. Если что-то не установится, Helm всё равно запустится — просто соответствующая функция будет недоступна (например, без `pywin32` не будет списка окон, но кнопки «свернуть всё» работают).

## ▶️ Запуск

1. **Задайте пароль администратора** (один раз). В папке проекта выполните:
   ```bash
   python set_password.py
   ```
   Программа спросит новый пароль и напечатает строку с хешем. Скопируйте её и вставьте в `core/config.py` в значение `_DEFAULT_PASSWORD_HASH`, либо задайте переменной окружения `HELM_PASSWORD_HASH` перед запуском. Этот пароль нужен для входа в админ-панель.

   > 💡 Проще всего сменить пароль уже после первого запуска — прямо в интерфейсе (Настройки → Смена пароля). Для самого первого входа можно временно использовать любой пароль из скрипта.

2. **Запустите сервер.** Дважды кликните по `run.bat`, либо в командной строке:
   ```bash
   python main.py
   ```

3. В окне появится адрес вида:
   ```
   ========================================================
   Helm запущен!
   Разделов: 7
   Открой на телефоне:  http://192.168.0.196:8765
   ========================================================
   ```

4. **Откройте этот адрес на телефоне** в браузере. Готово!

> 🔌 Чтобы сервер запускался автоматически при включении ПК, добавьте ярлык на `run.bat` в папку автозагрузки (`Win+R` → `shell:startup`).

## 📱 Как пользоваться с телефона

1. Убедитесь, что телефон и ПК в **одной Wi-Fi сети**.
2. Откройте в браузере телефона адрес из консоли (например, `http://192.168.0.196:8765`).
3. Вы сразу попадёте в **гостевой режим** — доступны разделы для гостей (по умолчанию Плеер, Монитор, Буфер обмена).
4. Чтобы получить полный доступ, нажмите **«Админ-панель»** вверху и введите пароль.

**Совет:** добавьте Helm на домашний экран телефона. В Safari (iPhone) — кнопка «Поделиться» → «На экран Домой». В Chrome (Android) — меню → «Добавить на главный экран». Тогда Helm будет открываться как обычное приложение.

## 👤 Роли: администратор и гости

Helm разделяет два типа пользователей:

### 🔑 Администратор (вы)
- Полный доступ ко всем 7 разделам.
- Доступ к настройкам, логам, списку онлайн.
- Вход: кнопка **«Админ-панель»** → пароль.

### 👥 Гости
- Заходят **без пароля** — сразу попадают в приложение.
- Видят **только разрешённые разделы** (настраивается в админке, по умолчанию Плеер / Монитор / Буфер обмена).
- При первом заходе могут указать своё имя — оно покажется вам в списке «кто онлайн».

**Безопасность ролей двухуровневая:** гость не сможет получить доступ к запрещённым разделам ни через интерфейс, ни напрямую по ссылке — сервер проверяет права на каждый запрос.

Какие разделы видят гости — настраивается галочками в **Настройки → Гостевые разделы**.

## 🎵 Токен Яндекс Музыки

Helm управляет музыкой через системный плеер Windows (SMTC) — **для этого токен не нужен**. Показ трека, обложки, пауза, переключение, перемотка, громкость работают сразу.

**Токен нужен только для одной функции — кнопки «лайк»** (добавить трек в понравившееся). Если она вам не нужна, можете пропустить этот раздел.

### Как получить токен

Токен проще всего получить через стороннюю программу **Get-Yandex-Music-Token**:

1. Откройте [github.com/Hazzz895/Get-Yandex-Music-Token](https://github.com/Hazzz895/Get-Yandex-Music-Token).
2. Перейдите на [страницу релизов](https://github.com/Hazzz895/Get-Yandex-Music-Token/releases/tag/v1.2) и скачайте **Release.zip**.
3. Распакуйте архив в любую папку.
4. Запустите **GetYandexMusicToken.exe**.
5. Пройдите обычную авторизацию в вашем аккаунте Яндекса.
6. После входа программа автоматически извлечёт токен из адресной строки и покажет его — **скопируйте** его.

> Как это работает: после входа Яндекс перенаправляет на служебные страницы завершения авторизации, и одна из них содержит токен в адресе. Программа просто достаёт его оттуда.

### Куда вписать токен

Есть два способа:

**Способ 1 — переменная окружения (рекомендуется, токен не попадёт в код).**
Перед запуском сервера задайте переменную `YANDEX_MUSIC_TOKEN`. Например, создайте файл `run_with_token.bat`:
```bat
@echo off
set YANDEX_MUSIC_TOKEN=сюда_вставьте_свой_токен
python main.py
```
и запускайте его вместо `run.bat`.

**Способ 2 — прямо в файле.**
Откройте `plugins/media/media_config.py`, найдите строку:
```python
_DEFAULT_YANDEX_TOKEN: str = ""
```
и впишите токен в кавычки:
```python
_DEFAULT_YANDEX_TOKEN: str = "ваш_токен"
```

> ⚠️ **Внимание:** токен даёт доступ к вашему аккаунту Яндекса. **Никогда не публикуйте его** и не коммитьте в открытый репозиторий. Если используете Способ 2 — не выкладывайте изменённый файл в открытый git.

## 🔐 Смена пароля

Пароль администратора можно сменить двумя способами:

- **Через интерфейс (проще):** Настройки → Смена пароля — применяется сразу, перезапуск не нужен.
- **Через скрипт:** `python set_password.py` напечатает хеш нового пароля; вставьте его в `core/config.py` (`_DEFAULT_PASSWORD_HASH`) или задайте переменной `HELM_PASSWORD_HASH`.

Пароль хранится в виде scrypt-хеша — в открытом виде он нигде не записывается.

## 🔒 HTTPS (защищённое соединение)

По умолчанию Helm работает по HTTP — для домашней сети этого достаточно. Но некоторые функции браузера (например, доступ к датчикам) требуют HTTPS.

Чтобы включить:
1. Настройки → Сервер → включите тумблер **HTTPS** → Сохранить.
2. **Перезапустите сервер.**
3. В логах при старте должно появиться `Схема подключения: https://...`.
4. Заходите на телефоне по `https://...`. Браузер предупредит о самоподписанном сертификате — это нормально для домашней сети, нажмите «Всё равно продолжить».

Сертификат создаётся автоматически (нужна библиотека `cryptography` или системный `openssl`).

## ⚙️ Настройки

Доступны администратору (Настройки — шестерёнка вверху):

- **Тема оформления** — авто / тёмная / светлая + выбор из 6 цветов.
- **Сервер** — порт, время жизни сессии, переключатель HTTPS (применяются после перезапуска).
- **Смена пароля** — сразу.
- **Гостевые разделы** — галочками выберите, что видят гости.
- **Кто сейчас онлайн** — список подключённых по именам.
- **Лог действий** — кто заходил, кто пытался войти в админку (включая неудачные попытки).

## 🧩 Как добавить свой раздел (плагин)

Helm построен на плагинах — каждый раздел это отдельная папка в `plugins/`. Чтобы добавить свой:

### 1. Создайте папку плагина
```
plugins/myplugin/
├── __init__.py
├── plugin.py       # логика
└── screen.html     # интерфейс раздела
```

### 2. Опишите плагин в `plugin.py`
```python
from pathlib import Path
from core.plugin_base import Plugin

_SCREEN = (Path(__file__).parent / "screen.html").read_text(encoding="utf-8")

class MyPlugin(Plugin):
    id = "myplugin"            # уникальный id (латиницей)
    name = "Мой раздел"        # название на плитке
    icon_svg = '<svg viewBox="0 0 24 24" ...>...</svg>'  # иконка

    def frontend(self) -> str:
        return _SCREEN

    async def handle_ws(self, msg: dict, send) -> None:
        # обработка команд с телефона
        cmd = msg.get("cmd")
        if cmd == "hello":
            await send({"type": "reply", "text": "Привет с ПК!"})

    async def start(self): ...   # опционально: запуск фоновых задач
    async def stop(self): ...    # опционально: остановка
```

### 3. Пропишите экспорт в `__init__.py`
```python
from .plugin import MyPlugin
__all__ = ["MyPlugin"]
```

### 4. Зарегистрируйте в `core/registry.py`
Добавьте в функцию загрузки:
```python
try:
    from plugins.myplugin import MyPlugin
    register(MyPlugin())
except Exception:
    log.exception("Не удалось загрузить мой плагин")
```

### 5. Интерфейс в `screen.html`
Раздел общается с сервером через глобальные функции оболочки:
```html
<button id="btn">Поздороваться</button>
<script>
(function(){
  // отправить команду на сервер
  document.getElementById('btn').addEventListener('click', () => {
    window.HelmSend({ plugin: 'myplugin', cmd: 'hello' });
  });
  // получать ответы
  window.HelmPlugins = window.HelmPlugins || {};
  window.HelmPlugins['myplugin'] = {
    onMessage: (msg) => {
      if(msg.type === 'reply') alert(msg.text);
    },
    onShow: () => { /* раздел открыт */ },
    onHide: () => { /* раздел закрыт */ }
  };
})();
</script>
```

После перезапуска новый раздел появится плиткой на главном экране. Ядро трогать не нужно — только эти 4 шага.

> 💡 Чтобы дать раздел гостям — отметьте его в Настройки → Гостевые разделы.

## ❓ Частые вопросы

**Телефон не открывает страницу.**
Проверьте, что телефон и ПК в одной Wi-Fi сети. Возможно, брандмауэр Windows блокирует — при первом запуске разрешите Python доступ к сети. Попробуйте временно отключить файрвол для проверки.

**Музыка не показывается / не управляется.**
Helm читает системный плеер Windows. Убедитесь, что играете музыку через приложение Яндекс Музыка (или любое, поддерживающее системные медиа-клавиши Windows).

**Список окон в разделе «Окна» пустой.**
Установите `pywin32`: `pip install pywin32`. Кнопки «свернуть/восстановить» работают и без него.

**Кнопка «лайк» не работает.**
Нужен токен Яндекс Музыки — см. раздел [Токен Яндекс Музыки](#-токен-яндекс-музыки).

**Можно ли пользоваться вне дома?**
Helm рассчитан на локальную сеть. Пробрасывать его в интернет не рекомендуется без дополнительной защиты.

## 🛡️ Безопасность

- Пароль хранится как scrypt-хеш, не в открытом виде.
- Гости не имеют доступа к запрещённым разделам — проверка на уровне сервера (и WebSocket, и HTTP).
- Раздел «Файлы» защищён от выхода за пределы дисков; системные пути скрыты.
- Helm не передаёт никаких данных наружу — всё работает локально в вашей сети.
- ⚠️ Не пробрасывайте Helm в открытый интернет без понимания рисков. Проект задуман для домашней Wi-Fi сети.
